## THE CORE LOOP

```
  YOU OBSERVE           YOU DOCUMENT           YOU IDEATE             YOU REPEAT
  ─────────────         ─────────────          ─────────────          ─────────────
  A broken UPI    →     Rich text editor  →    Attach ideas    →      Tomorrow:
  A long queue          Structured fields       Link to problems        10 more
  A missing app         Auto-stamped            Many-to-many            problems
  An ignored gap        Auto-categorised        Full context            No. Excuses.
```

## WHY THIS EXISTS

Most people scroll past friction. The hospital queue? *Normal.* The broken payment flow? *Whatever.* The missing product that nobody built yet? *Somebody will figure it out.*

**Founders can't afford to think like that.**

This app is built on one belief — **problem-spotting is a muscle.** It atrophies when you ignore it. It sharpens when you train it daily. Inspired by [James Altucher's Idea Machine](https://jamesaltucher.com/blog/the-ultimate-guide-for-becoming-an-idea-machine/), FounderLog replaces vague idea generation with something more rigorous: **structured, timestamped, categorised problem observation.**

Write 10 problems a day. Every single day. Watch your founder instincts become unreasonably sharp.

## FEATURE BREAKDOWN

<br/>

| | Feature | What It Does |
|:---:|:---|:---|
| ⚡ | **Quick Capture** | Long-press the main button → one-line instant capture. Polish it later. |

| 📊 | **Full Analytics** | Weekly charts, monthly heatmap, domain breakdown, status bars — see your growth objectively. |
| 🌑 | **Dark Only** | Deliberately and unapologetically dark. Glassmorphism. Amber accents. No light mode. |

## DESIGN LANGUAGE

> *"Think boldly. Do not default to generic patterns. Every screen must have a deliberate, thoughtful design decision behind it."*

<br/>

| Principle | Implementation |
|:---|:---|
| **Dark Only** | Deep charcoal `#0A0A0F` base. No light mode. This is a focus tool, not a lifestyle app. |
| **Electric Amber** | `#FFB800` accent — bold, energetic, demands attention without screaming. |
| **Glassmorphism** | Frosted glass cards with `rgba(255,255,255,0.06)` and glowing borders. |
| **Living UI** | Cards are not static. Status edges pulse, breathe, and glow based on problem state. |
| **Intentional Type** | Oversized headings. Generous letter-spacing. Words carry weight. |
| **Tactile Feedback** | Haptic responses on every meaningful interaction. The app feels physical. |
| **Purposeful Motion** | Rings fill. Numbers count up. Cards breathe. Nothing animates without reason. |

<br/>

## CONTRIBUTING

Contributions are welcome. But read this first.

This app has a **design philosophy**. We do not do generic. We do not do default Material UI. Every button, every animation, every color choice is deliberate. If you are contributing UI changes, understand the design language before touching a single component.

```bash
# Fork → Branch → Build → Commit → PR

git checkout -b feature/your-feature-name
git commit -m "Add: clear description of what you built"
```
